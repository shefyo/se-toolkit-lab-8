## What is a protocol

A protocol (communication protocol) is a set of rules that define how data is transmitted between two parties (typically a [web client](./web-infrastructure.md#web-client) and a [web server](./web-infrastructure.md#web-server)). Different protocols suit different communication patterns.

Docs:

- [Web protocols](https://developer.mozilla.org/en-US/docs/Web/Reference/Protocols)

## `HTTP`

[`HTTP`](./http.md#what-is-http) is the standard protocol for [web APIs](./web-api.md#what-is-a-web-api). The [client](./web-infrastructure.md#web-client) sends a request and the [server](./web-infrastructure.md#web-server) sends one response. Communication is always initiated by the client.

Used for: [`REST` APIs](./rest-api.md#what-is-a-rest-api), file downloads, page loads.
